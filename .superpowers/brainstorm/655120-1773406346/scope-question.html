Admin Panel — Who Uses It?

This determines auth, security, and complexity level

A

Single Admin (You Only) (Recommended)

No login needed — access via kubectl port-forward. Simplest approach, zero auth code. The panel is a private tool behind K8s network.

B

Small Team (2-5 Admins)

Basic auth (password from Key Vault) or Azure AD login. ClusterIP service, accessed via port-forward or private ingress.

C

Self-Service Portal

Users create their own agents with Feishu credentials. Needs auth, RBAC, quotas, public ingress + TLS. Much more complex.
