Admin Panel — Scope & User Model

First question: Who uses this panel and what can they do?

Who will use this admin panel?

This determines auth requirements, security posture, and UI complexity.

A

Single Admin (You Only)

No login needed — just port-forward to access. Simplest approach, zero auth overhead. The panel is a private tool only accessible via kubectl port-forward.

B

Small Team (2-5 Admins)

Basic auth (username/password from Key Vault) or Azure AD login. Panel runs as a ClusterIP service, accessed via port-forward or private ingress.

C

Self-Service Portal

Users can create their own agents with their Feishu credentials. Needs proper auth, RBAC, quotas, and public-facing ingress with TLS. Significantly more complex.
